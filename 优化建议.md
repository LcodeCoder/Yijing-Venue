# 《场地弈境》优化建议

> 基于当前 MVP 实现（Spring Boot + Vue 3）的产品与玩法评估整理。  
> 目标：从「能玩的完整对局」提升到「愿意连开、愿意安利」的可玩度与完成度。  
> 日期：2026-07-19

---

## 1. 总体评估

### 1.1 一句话结论

作为原创**控场卡牌** MVP，骨架扎实、对局闭环完整，可玩度约 **7/10**。  
主要短板不在「有没有核心玩法」，而在：**文案与规则不一致、上手成本、AI 深度、卡牌视觉记忆、长线内容厚度**。

### 1.2 已具备的优势

| 维度 | 现状 |
|------|------|
| 核心差异化 | 无生命值，围绕场地归属与积分/绝杀 |
| 决策密度 | 灵力不跨回合、每场最多 2 单位、射程限制 |
| 双重胜利 | 稳定控场绝杀 vs 终局积分结算 |
| 对局反馈 | 抽牌揭示、争夺演出、部署飞行、先手骰、阶段计时 |
| 产品闭环 | AI / 好友房 / 天梯、图鉴、卡组、规则、档案 |

### 1.3 当前拖累可玩度的点

1. **首页仍写「五域 / 5 场地 / 3 灵力 / 8 回合」**，实际为 3×3～5×5 可变棋盘，口径冲突。
2. **AI 偏贪心脚本**（优先打守力最低目标），熟练后易被摸透。
3. **单位永不阵亡**有特色，但冲击感偏弱，中后期易变成「换归属」。
4. **卡面以符号+渐变为主**，收集欲与传播力不足。
5. **规则信息多**（阶段、射程、壁垒、初始部署、骰先手），第一局仍偏试错。

### 1.4 分人群可玩度

| 人群 | 可玩度 | 说明 |
|------|--------|------|
| 策略 / 棋牌爱好者 | 高 | 能连打找组合 |
| 休闲 5 分钟爽一把 | 中偏低 | 门槛 + 60 秒阶段压感 |
| 追求长期构筑 / 赛季 | 中 | 系统有，卡池与 AI 深度不够 |

## 2. 文案与产品一致性（P0）

首页、规则页、README、战场必须同一套规则叙事。

### 2.1 建议统一的对外口径

| 项目 | 建议表述 |
|------|----------|
| 棋盘 | 可选 3×3 / 4×4 / 5×5 域面 |
| 灵力 | 每回合灵力 = 棋盘边长（3 / 4 / 5），不跨回合累计 |
| 回合 | 总回合约 = 边长 × 3（如 3×3 为 9 回合） |
| 胜利 | ① 控制过半场地并连续两次结算触发绝杀；② 否则比累计积分 |
| 阶段 | 部署 / 争夺各限时 60 秒（可再分模式调整） |

### 2.2 具体改动清单

- [x] 首页 Hero 统计：去掉「5 固定场地 / 3 灵力 / 8 回合」，改为与上表一致。
- [x] 首页文案「五域主动权」改为「九域 / 域面主动权」或中性表述。
- [x] 首页示意图：从五格菱形改为默认 **3×3 九宫示意**。
- [x] `README.md` 与规则页、实现逻辑三方对齐（避免文档写五格、代码跑九格）。
- [x] 规则手册与战斗内引导用词统一（部署、争夺、绝杀进度、射程）。

**预期收益：** 降低误导，提升专业感与第一印象信任。

## 3. 样式与体验建议

### 3.1 设计原则（延续 PRODUCT.md）

1. 展示因果：部署、争夺、抽牌、弃牌在空间与视觉上可追踪。
2. 合法行动明显：费用、目标、射程、容量在提交前可见。
3. 大场面留给关键事件：夺场、绝杀、抽 SSR；日常控件保持轻快。
4. 战场可读优先于装饰：归属、邻接、单位槽、当前阶段 > 光效。
5. 颜色不单独承担信息：归属用色 + 角标/描边/图标双编码。

### 3.2 战场可读性

| 建议 | 说明 |
|------|------|
| 邻接 / 射程连线 | 选中单位时，用淡色「灵脉线」连到射程内场地 |
| 归属双编码 | 己 / 敌 / 空 角标或徽章，不只靠绿红 |
| 核心场地强化 | 缓慢脉动；结算时飘字「积分 ×2」 |
| 绝杀进度主信息 | 「绝杀 1/2」与比分同级展示，打断即重置要可读 |
| 临时状态图标 | 封印、战力+、守力+、射程+ 常驻在单位上 |
| 非法操作反馈 | toast 写清：超出射程 / 灵力不足 / 阶段不对 / 已行动 |
| 移动端详情 | 场地效果被折叠时，长按弹出完整效果（对齐卡牌详情） |

### 3.3 卡牌视觉

不必立刻上精细插画，建议「低成本统一模板」：

- **场地**：地形纹章构图  
- **单位**：剪影角色构图  
- **术式**：符阵构图  
- **秘策**：星图构图  

稀有度边框分层：C 素雅 → R 金线 → SR 辉光 → SSR 紫雾。  
抽牌演出：SSR 单独加长约 0.3s + 独立音效，强化稀缺感。

### 3.4 整体气质

- 大厅：可偏仪式感、雾海神秘。  
- 战场：偏冷静战术台，减少连续 overlay 造成的视觉疲劳。  
- 尊重 `prefers-reduced-motion`：长演出可降为短淡入。

### 3.5 音效与 Game Feel（分层）

| 层级 | 事件 | 反馈建议 |
|------|------|----------|
| 小 | 点牌、合法目标 hover | 轻 tick |
| 中 | 部署落场、单位驻扎 | 短音 + 轻微 scale pop |
| 大 | 夺场、壁垒第二次突破、绝杀 | 音效 + 短 hit-stop + 轻震 |

原则：反馈短暂后回落；重要性越高 juice 越强，避免每一步都全开。

## 4. 玩法优化建议

### 4.1 核心方向（不要偏航）

游戏定位是 **「空间争夺的卡牌棋」**，而不是「更大伤害的换血卡」。  
后续深度应优先堆：

- 连片 / 几何价值  
- 单位调防与机动  
- 绝杀压力与打断  
- 清晰流派目标  
- 可读的失败原因  

**不建议：** 引入完整血量击杀当主轴、复杂响应连锁、默认超大棋盘、过量纯随机。

---

### 4.2 每局要有「主线压力」

避免整局只有「铺场 → 换归属 → 拖积分」。

| 思路 | 做法 | 目的 |
|------|------|------|
| 绝杀预警 | 控场 ≥ 过半时顶部提示「再结算 1 次将绝杀」 | 落后方必须反击 |
| 分差阈值 | 分差过大时强化翻盘手段（可与现有天平秘策联动） | 减少无效尾盘 |
| 回合章节感 | 前中期扩张 → 中期冲突 → 终局 2 回合必决策 | 节奏有段落 |

---

### 4.3 强化棋盘几何

| 机制 | 建议数值（初稿，需实装后微调） | 作用 |
|------|--------------------------------|------|
| 邻接协同 | 己方两块场地相邻：结算各 +0～1 分，或守力 +1 | 鼓励连片 |
| 夹击 | 目标被 ≥2 个己方邻接场地夹住：争夺战力 +1 | 进攻有预谋 |
| 路径/地形 | 部分场地使相邻判定距离 +1 | 壁垒不只「打两次」 |
| 边角特性（可选） | 如边陲：部署费 -1 但积分为 0 | 开局落点有流派 |

原则：每一步部署都在改「下一回合谁够得着谁」。

---

### 4.4 单位永不阵亡 → 用状态补张力

保留永不阵亡特色，用状态制造起伏：

| 状态 | 效果 | 解除 |
|------|------|------|
| 疲惫 | 本回合已争夺 / 被压倒性逼退 | 下回合自动 / 刷新术式 |
| 动摇 | 所在场被夺后，下次守力 -1 或不可主动攻 | 1 回合 |
| 扎根 | 同场连续驻守 2 回合：守力 +1 | 撤离或被夺场清零 |
| 行军（可选） | 部署当回合不可攻 | 防「落子秒抢」过猛 |

---

### 4.5 资源层：灵力之外的软资源（三选一即可）

| 方案 | 简述 |
|------|------|
| 气势 | 成功争夺 +1，失败 -1；满层可免费 1 费术式或本回合射程 +1 |
| 域印 | 每控制一场结算获 1，用于小幅升级场地或推动秘策 |
| 筛牌 | 每回合限 1 次：弃 1 抽 1（0～1 灵力） | 降低烂手绝望 |

目标：玩家有时会选择「少铺一张场，换一次筛牌/气势」。

---

### 4.6 双胜利条件拉扯

1. **绝杀必须「稳」**  
   - UI 主展示进度 1/2。  
   - 第二次结算时若仍过半才触发；中途被打掉则重置（若规则已如此，确保表现层一致）。

2. **积分路线反制绝杀路线**  
   - 场少分高时，镜潮 / 天平类效果更有价值。  
   - 慎用「花分抵消绝杀」等强规则，需严格数值验证。

3. **终局回合特殊规则（可选）**  
   - 最后一回合：不可新部署场地，只能争夺与术式 → 纯读局。

---

### 4.7 部署阶段的微决策（优先两项）

1. **单位调防**  
   花费 1 灵力，将单位移动到相邻己方场地（不发动争夺）。  
   → 动态补洞，比只能撤退进弃牌更有棋感。

2. **场地覆盖 / 改造**  
   对己方场再打场地卡：替换效果或叠一层。  
   → 中后期场地手牌不至于完全废牌。

其他可选：

- 主动弃牌有小收益（弃场地抽 1 / 弃单位本回合射程 +1）。  
- PVP 可考虑「宣告进攻方向换小加成」的心理战（非优先）。

---

### 4.8 构筑与流派

建议明示 **3～5 套官方原型**（比闷头加卡更重要）：

| 流派 | 牌感关键词 | 胜利路径 |
|------|------------|----------|
| 壁垒龟 | 玄岩壁垒、高守单位、固守 | 拖满回合拼分 / 防绝杀 |
| 游猎射 | 斥候、夜航游侠、折界通路、星灯塔 | 跨距偷核心、拆连片 |
| 锻场突 | 赤焰锻场、决斗家、奔袭者、灵潮 | 压倒性加分滚雪球 |
| 运营抽 | 典藏馆、先知、星图洞见 | 过牌找关键键 / 秘策 |
| 绝杀快控 | 低费场、铺量、万域归一 | 中期过半连结算 |

配套：

- [ ] 新手直接赠送 3 套完整主题卡组（替代单一 40 张大杂烩体感）。  
- [ ] 图鉴为关键牌标注「适合流派」。  
- [ ] 构筑约束写清：同名上限、SSR 上限、场地/单位最低张数等。

---

### 4.9 新手体验

1. **可选或强制 3 分钟教程局**（固定牌序 + 脚本 AI）  
   - 只教：放场地 → 放单位 → 选单位点敌场。  
   - 第二局再讲壁垒、射程、绝杀。  
2. **默认推荐 3×3**；4×4 / 5×5 标为「进阶」。  
3. **非法操作必须可读**（见 3.2）。  
4. 人机阶段时限可放宽到 90 秒，排位保持 60 秒（或首回合部署 90 秒）。

---

### 4.10 AI 与单机内容

#### AI 难度分层

| 难度 | 行为要点 |
|------|----------|
| 入门 | 现有贪心逻辑即可 |
| 标准 | 优先抢/保核心；会补防；会用封印打断高攻 |
| 困难 | 估算下回合绝杀威胁；留灵力打 RANGE+突袭；壁垒二次连破 |

实现上优先 **威胁分 + 目标权重**，不必上完整搜索树。

#### 单机内容包装

- **试炼关卡**：教放场 → 教射程 → 破壁垒两次 → 防绝杀。  
- **残局谜题**：给定局面，「3 灵力内夺取核心」——规则清晰，适合本品。  
- **AI 人格**：雾隐（稳守核心）、裂隙（高攻）、镜潮（反制分）——同一规则不同权重。

AI 回合客户端动画总时长建议控制在约 **8～12 秒/回合** 体感，避免干等。

---

### 4.11 随机性边界

已有：抽牌、先手骰。建议原则：

- 骰子只决定先手，不直接决定胜负。  
- 烂手时提供**有限**筛牌/寻牌，而不是加大暴击类随机。  
- 玩家输了应能归因：「射程断了 / 绝杀没防住 / 连片被拆」，而非「今天牌背」。

## 5. 实施优先级

### 5.1 两周冲刺建议（性价比）

| 优先级 | 事项 | 预期收益 |
|--------|------|----------|
| **P0** | 统一首页 / 规则 / 示意图与真实 3×3 规则 | 专业感、信任 |
| **P0** | 新手引导局 + 非法操作明确反馈 | 第一局完成率 |
| **P0** | 绝杀进度 UI 化，打断重置可读 | 中盘紧张感 |
| **P1** | AI 标准/困难（保核心、破绝杀、补防） | 复玩 |
| **P1** | 射程邻接连线 + 单位状态图标 | 策略可读 |
| **P1** | 单位相邻调防（1 灵力）+ 邻接小加成 | 棋感 |
| **P2** | 卡牌简易纹章插画体系 | 收藏与传播 |
| **P2** | 三套官方主题卡组 + 周常小规则 | 长线内容 |
| **P2** | 试炼关 / 残局谜题 | 单机寿命 |

### 5.2 若只做 3 个玩法改动

1. **绝杀进度 + 威胁表现**（含打断重置）。  
2. **单位可花灵力挪到相邻己方场 + 邻接小加成**。  
3. **三套官方流派卡组 + 标准 AI 会保核心/防绝杀**。

## 6. 建议的验收标准

### 6.1 体验验收

- [ ] 从未接触的玩家可在 **3 分钟内**完成第一次合法争夺。  
- [ ] 首页与规则描述与对局内数值 **零冲突**。  
- [ ] 绝杀临近时，双方都能在 UI 上 **不查规则** 理解风险。  
- [ ] 标准 AI 下，熟练玩家胜率不再接近 100%（目标例如 60%～75% 区间，可再调）。  
- [ ] 至少 **3 种**可描述的取胜套路（龟、射、突/绝杀）在实战中都成立。

### 6.2 技术与表现验收

- [ ] 合法目标、超出射程、费用不足均有即时文案。  
- [ ] 关键状态（封印、增幅、射程、壁垒突破次数）在场上可见。  
- [ ] 减少动态偏好下，对局仍可完整进行且信息完整。  
- [ ] AI 回合不造成长时间无操作黑屏（动画可跳过或总时长可控）。

## 7. 明确不做或慎做

| 方向 | 原因 |
|------|------|
| 以击杀单位 / 扣血为主轴 | 稀释「归属」特色 |
| 默认 6×6 或更大 | 信息过载，3×3 应是门面 |
| 复杂响应窗口 / 长连锁 | 网页操作与理解成本过高 |
| 大量隐藏私有信息 | 与「局势可读」原则冲突 |
| 为深度而堆无协同新卡 | 多样性来自约束与目标，不来自卡量 |
